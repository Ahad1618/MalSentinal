# 🛡️ MalSentinel — AI-Powered Malware Detection & Classification System

> CS 2005: Artificial Intelligence — Group Project  
> Abdul Ahad (23P-0625) · Muhammad Owais (23K-5544)

Static malware detection using **LightGBM**, **Bayesian Inference**, and **SHAP** explainability — trained on the EMBER 2018 dataset.

---

## Features

- **Static Analysis** — analyzes `.exe` / `.dll` files without executing them
- **LightGBM Classifier** — gradient boosting on 2,381 EMBER PE features (AUC ~0.99)
- **Bayesian Network** — Naive Bayes on dangerous API imports with per-node probability display
- **SHAP Explanations** — waterfall chart showing which features drove each prediction
- **Beautiful Dark UI** — Streamlit dashboard with progress animation, metric tiles, and tabbed results
- **PDF Report** — auto-generated downloadable analysis report

## Algorithm Details

### LightGBM (Primary Classifier)
Trained on 2,381 numerical features from the EMBER 2018 feature set:
- General file info, COFF/Optional header fields
- Section table statistics (entropy, size, characteristics)
- Import/export feature hashing (DLL names, DLL:Function pairs)
- Byte frequency histogram (256 bins)
- Byte-entropy joint histogram (256 bins)
- Printable string statistics

### Bayesian Network (Naive Bayes)
Gaussian Naive Bayes on the import-table hash region. Network nodes:
- `VirtualAlloc` — memory allocation (injectors)
- `CreateRemoteThread` — remote code execution
- `WriteProcessMemory` — process memory manipulation
- `OpenProcess` — process handle acquisition
- `URLDownloadToFile` — network dropper
- `IsDebuggerPresent` — anti-analysis
- `StringEntropy` — obfuscated string indicator

Computes `P(Malicious | API features)` as an independent second opinion.

### SHAP Explanations
TreeExplainer provides per-prediction Shapley values showing each feature's contribution toward the malicious or benign verdict.
